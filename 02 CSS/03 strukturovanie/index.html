spravne strukturovat html - používať elementy na čo sú určené, nepoužívať na všetko div-y, pouzivat tagy nav, header, footer, article je mozne pouzivat iba div-y a nastylovat ich, no nie je to potom prehladne napr. pouzitie nav namiesto div je vyhodne, pretoze screen reader obsahuje moznost necitat ich. ak by to bol div (a nie nav), tak by ich pre zrakovo znevyhodnenych cital zakazdym, bez moznosti zvolit si, ze navigacne menu nechcem mat precitane stale, stacilo raz.
html sa nema starat o vizualnu stranku, tu kontrolujem cez CSS napr. nerobiť < br > < br > do odseku, keď stačí pridat dalsi odsek, ktoremu viem nastavit margin aky potrebujem
